JavaScript Loading & Execution Methods

Understanding how JavaScript files load and interact with HTML

Your Code Example

You're using a module script tag to load your JavaScript:

<script type="module" src="/js/main.js"></script>

This tells the browser to load main.js as an ES6 module. Modules have their own scope and use strict mode by default.

1. Standard Script (Synchronous)

HTML parsing stops until the script is downloaded and executed.

<script src="script.js"></script>

2. Async Script

Script loads in parallel with HTML parsing, executes immediately when ready.

<script async src="script.js"></script>

3. Defer Script

Script loads in parallel but executes after HTML parsing is complete.

<script defer src="script.js"></script>

4. Module Script (Your Method)

ES6 modules are deferred by default. They have their own scope and support imports/exports.

<script type="module" src="module.js"></script>

Key Concepts

DOMContentLoaded vs Load Events

DOMContentLoaded: Fires when HTML is fully parsed and DOM is built

Load: Fires when all resources (images, stylesheets) are loaded

Dynamic Script Loading

You can also load scripts dynamically using JavaScript:

const script = document.createElement('script'); script.src = 'dynamic.js'; document.head.appendChild(script);

About Localization & Internationalization

This page demonstrates internationalization (i18n) and localization (l10n) by supporting multiple languages. Key implementation details:

  • Text content is stored in language-specific JSON objects
  • HTML elements have data-i18n attributes for text replacement
  • CSS handles RTL (right-to-left) layout for Arabic
  • Language switching updates all text and layout direction
